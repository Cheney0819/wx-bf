# Android 启动图标优化设计

## 目标

将“捉捉虫”的 Android 桌面启动图标替换为用户提供的蓝发桌宠角色，使其在常见 Android 启动器的图标遮罩下保持清晰、居中且易识别。

## 已确认的视觉设计

- 主体：用户提供的蓝发桌宠角色图。
- 背景：纯白色。
- 处理：移除与画布边缘连通的白色底图，保留角色本身的白色衣物与高光。
- 层次：角色置于启动图标安全区中央，保留轻微投影以保证白色部位的轮廓可见。
- 风格：保留原始像素/二次元画风，不重绘角色或添加文字、徽章。

## Android 资源设计

- 新建自适应图标：前景层使用处理后的角色 PNG，背景层使用纯白色。
- `android:icon` 与 `android:roundIcon` 都改为新的 mipmap 自适应图标资源。
- 现有 `ic_pet_notification` 继续用于前台服务通知的小图标，不与桌面图标共用，以避免通知栏兼容性退化。
- 前景资源按 Android 自适应图标安全区留白导出，降低圆形、圆角方形与厂商自定义遮罩裁切角色的风险。

## 验证

- 执行 Android debug 构建，确保资源链接成功。
- 检查合并后的 `AndroidManifest.xml`，确认桌面图标和圆形图标均指向新资源。
- 检查导出的图标前景和最终预览：白色背景、角色完整、无白底矩形残留。

## 范围

仅更新应用桌面启动图标及其资源引用；不改变应用名称、界面、桌宠动画或通知图标。
